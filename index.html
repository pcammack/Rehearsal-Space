REHEARSAL SPACE v6.6
WHAT'S NEW IN v6.6
· Passage queue: Checkbox on each passage to build a play queue. ▶ Play queue button runs through checked passages in chronological order, one pass each.
· Loop counter: Visible count of consecutive loop plays with a reset button. Appears when a loop is active.
· Input level monitor: Mic level bar appears when recording starts — confirms signal before committing to a take.
· Single-pass recording: ⊳ Single button next to Record — plays through once without looping.
· Clip speed badge: Recording tempo shown on each take in the Takes panel.
· Session focus prompt: "What are you listening for today?" field at the top of Notes, stored per piece.
· seekClip fix: Clip mini-player now correctly resumes at the recorded speed after seeking.
· Blend system removed: Live playback blend and its UI fully cleaned out. WAV download unchanged.
· Performance marks never autonumber: Adding a performance mark no longer triggers renumbering of structural marks. Autonumbering and cascade are now scoped exclusively to structural marks (type R).
· Recordings stay in sync at non-100% speed: Clips now store the playback rate at which they were recorded. The mini-player, mixer, and blend reference all match that speed, so a take recorded at 75% reviews correctly at 75%.
· Manual piece data save/load: Three new buttons in the Repertoire panel — ↓ Save piece (exports current markers + notes to a dated JSON), ↑ Load piece JSON (restores markers and passages from a saved file), and ↑ Load notes .txt (loads notes from a plain text file). Useful for data recovery and version protection.
FUTURE FEATURES
NEAR-TERM — contained, low risk
· Input level monitor — mic level indicator to verify external mic is active before recording
· Record segment without looping — single-pass immediate recording, no loop required
· Score-to-marker workflow — tap-to-place guided mark placement from a rehearsal letter list
· Progress log — recording history per piece with date/tempo; compare clips across sessions
· Folder migration — on reconnecting to Rehearsal Space folder, scan for legacy notes.txt and merge into piece notes (run once per piece)
MEDIUM — moderate complexity
· Spool — curated sequence of non-contiguous passages at stepped tempos, plays through automatically; option to record one full pass as a single clip
· Clip speed label — display recording tempo next to each take (e.g. "I.1 · 75%") as a permanent reminder
· Latency calibration refinement — quality scoring, per-session variance tracking over time
· Highlight key areas — visual passage emphasis on waveform (colour-tinted region between marks)
LONGER HORIZON — architectural
· Standalone cloud app — self-contained app hosted on a permanent URL; stores audio and piece data remotely (e.g. Cloudflare R2 or similar), no local files required, fully portable across devices. Would enable a significantly simpler interface since file management disappears. Prerequisite for the iPhone version below.
· iPhone companion app — simplified mobile version (separate app, same URL/library); loads pieces from the existing library, plays passages, records takes. No mark editing, no library additions, no woodshed. Fully compatible with the desktop data format. Contingent on the standalone cloud architecture above.
· ForScore integration — import rehearsal mark list from ForScore export for tap-to-place workflow
NOTIONAL — ideas only, not committed
· Metronome with time-signature structure — define meter and tempo changes per section; useful for building custom reference recordings for solo repertoire without a commercial recording
· Basic tuner
· Multi-device sync via export-to-URL or shared cloud storage
WORKFLOW
1. Select a piece from the Library — the file picker opens automatically.
2. Press Play. Press M to drop markers while listening.
3. In Loops: pick Start → End markers, click Add. Click ⟳ to loop, ▶ to preview.
4. Add notes in the Notes panel — saved automatically.
5. Save via Library → + Save current.
MARKER NAMING
· Numeric: 1 → 2 → 3
· Alpha: A → B → C
· Dash-numeric: A-2 → A-3 → A-4 (subsections)
· Dash-alpha: 3-A → 3-B → 3-C (subsections)
· Placeholder: . (period) — excluded from cascade and loop dropdowns
